配置字段详解

You only need api_key; add others on demand

Hot reload: 5s
api_key
Your custom Bearer key. Clients must send Authorization: Bearer <api_key>
proxy
Explicit proxy, e.g. http://127.0.0.1:7890. Falls back to env vars if empty.
cookies
Gemini web Cookie. Helps fetch/refresh tokens more reliably.
gemini_url / gemini_home_url
Override endpoints for reverse-proxy setups only.
tokens - reserved field, not used in current code.

鉴权与会话

固定 X-Session-ID 以保持多轮对话

conversation_id > header
Auth
Your api_key protects this proxy only, not a Google key.
Session
Fix X-Session-ID per user/conversation to reuse Gemini context.
Alternative - pass conversation_id in the JSON body (higher priority than header).

curl 调用示例

Copy & paste ready; Base URL auto-filled

Use SDK
health check 
curl -s "{{BASE}}/api/telemetry"
models 
curl -s "{{BASE}}/v1/models"
chat · non-stream 
curl -s "{{BASE}}/v1/chat/completions" \
  -H "Authorization: Bearer change-me" \
  -H "Content-Type: application/json" \
  -H "X-Session-ID: demo-session" \
  -d '{"model":"gemini-3-flash","stream":false,"messages":[{"role":"user","content":"你好"}]}'
chat · stream (SSE) 
curl -N "{{BASE}}/v1/chat/completions" \
  -H "Authorization: Bearer change-me" \
  -H "Content-Type: application/json" \
  -H "X-Session-ID: demo-session" \
  -d '{"model":"gemini-3-flash","stream":true,"messages":[{"role":"user","content":"用三句话介绍一下你自己"}]}'

流式 SSE 说明

stream=true → chunked SSE, ends with [DONE]

curl -N
Format
Each chunk is data: { ... }; final message is data: [DONE].
Proxy + Streaming
Unstable proxies may break the stream. Check connectivity first, then lower concurrency.

排障速查

Log level + proxy + cookies are your first 3 checks

Security
HTML / Captcha / Blocked
Gemini anti-bot. Fix: change IP/proxy → lower concurrency → add cookies.
Timeout / Connection refused
Check proxy; without it, check system env vars.
Conversation breaks
Fix X-Session-ID - don't use random UUIDs per request.
Need verbose logs
Set log_level to debug, then reproduce.

安全建议

Don't commit credentials; expose only to trusted networks

Back to top

Limit exposure
Bind to localhost / internal network. For public: reverse proxy + auth + TLS.
