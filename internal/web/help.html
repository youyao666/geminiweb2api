部署前准备

先确认程序、配置文件和访问方式，避免把基础问题带到接口层。

Binary First
你需要准备什么可执行文件、同目录的 config.json、可用网络环境，以及你自定义的 api_key。
建议先做什么优先确认 /api/telemetry 能返回 JSON，再开始接 SDK 或 curl。

快速开始

目标是让第一次接入的用户在几分钟内跑通完整链路。

4 Steps
1编辑配置先设置 api_key。只有需要代理或 Cookie 时再补其他字段。
2启动程序Windows 可双击程序，命令行也可直接运行可执行文件。
3检查服务访问 /api/telemetry，确认状态与 uptime 正常。
4发送请求向 /v1/chat/completions 发 OpenAI 兼容请求，并带上 Bearer Key。
config.json 示例
{
  "api_key": "change-me",
  "proxy": "",
  "cookies": "",
  "log_level": "info",
  "note": ["geminiweb2api running"]
}

配置项说明

只解释真正影响交付的字段，避免堆砌技术噪声。

Minimal
api_key对外暴露给客户端的 Bearer Key，请求头使用 Authorization: Bearer <api_key>。
proxy只有网络环境需要代理时才填写，例如 http://127.0.0.1:7890。
cookies用于 Gemini Web 登录态场景，可提高 token 获取和刷新稳定性。
log_level排错时建议临时切到 debug，恢复后再切回 info。

多账号号池

当前版本已经支持多账号、会话绑定、轮询调度和失败指数避退。

Pool
会话绑定同一个 X-Session-ID 会优先继续走同一个账号。
轮询分配新会话会在健康账号之间按权重轮询分配。
指数避退账号失败后会进入 30 秒起步、最高 30 分钟的 backoff。
accounts 配置示例
{
  "api_key": "change-me",
  "accounts": [
    {
      "id": "acc-1",
      "email": "first@example.com",
      "cookies": "SID=...; APISID=...",
      "token": "",
      "enabled": true,
      "weight": 1
    },
    {
      "id": "acc-2",
      "email": "second@example.com",
      "cookies": "SID=...; APISID=...",
      "token": "",
      "enabled": true,
      "weight": 1
    }
  ]
}

接口总览

把真正常用的入口集中讲清楚。

API
GET /api/telemetry查看服务在线状态、请求数、RPM 和 token 统计。
GET /api/accounts查看账号池与当前绑定状态，需 Bearer Key。
POST /api/accounts新增或更新账号，需 Bearer Key。
POST /api/accounts/{id}/refresh立即刷新某个账号的 token 状态。
GET /v1/models适合 SDK 初始化、探活或模型发现。
POST /v1/chat/completions主要聊天入口，兼容 OpenAI Chat Completions。

请求示例

所有示例会自动带入当前页面的 Base URL。

chat completions
curl -s "{{BASE}}/v1/chat/completions" \
  -H "Authorization: Bearer change-me" \
  -H "Content-Type: application/json" \
  -H "X-Session-ID: demo-session" \
  -d '{"model":"gemini-3-flash","stream":false,"messages":[{"role":"user","content":"你好"}]}'
Node.js
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "change-me",
  baseURL: "{{BASE}}",
});

const resp = await client.chat.completions.create({
  model: "gemini-3-flash",
  messages: [{ role: "user", content: "你好" }],
});

console.log(resp.choices[0]?.message?.content);
Python
from openai import OpenAI

client = OpenAI(
    api_key="change-me",
    base_url="{{BASE}}",
)

resp = client.chat.completions.create(
    model="gemini-3-flash",
    messages=[{"role": "user", "content": "你好"}],
)

print(resp.choices[0].message.content)

故障排查

把常见问题先定位到网络、鉴权、登录态和会话这几个方向。

Troubleshoot
请求 401确认调用接口时带了 Authorization: Bearer <api_key>，并且 key 与 config.json 一致。
Gemini 无响应或超时先检查 proxy 是否有效，再看网络环境是否能正常访问 Gemini。
页面态抓取失败优先补充有效的 cookies，必要时也可以临时填写手动 token。
多轮对话不连续保持同一个 X-Session-ID，让服务端能维持账号绑定和上下文连续性。
